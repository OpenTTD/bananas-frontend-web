Since 27 April 2020, BaNaNaS changed the authentication method to GitHub. If you uploaded content before that time, you need to migrate the author of that content once from your old OpenTTD account to a new GitHub account. The following steps help you make this happen. You only need to do this once for all your content.

First, create a GitHub account if you don't already have one: github.com

Next, you have one of two options. You only need to do one of them, and only for one of your content. We will automatically migrate all your other content for you.

Option one - TT-Forums (preferred)

  • Is the URL of any of your content linking to tt-forums.net?
  • Are you the topic starter of this thread?

If so, fill in the next template, and send it to both frosch and TrueBrain via TT-Forums private message.

Subject: BaNaNaS author migration 
Hello,

I would like to migrate my BaNaNaS account to GitHub.

Link to package on BaNaNaS: <copy here the URL of your content: https://bananas.openttd.org/package/***/*** >
GitHub profile page: <copy here the URL of the GitHub profile page: https://github.com/*** >

Option two - email

  • You still have access to the email-address you used to register your OpenTTD account?

If so, fill in the next template, and send it to info@openttd.org via email.

Subject: BaNaNaS author migration 
Hello,

I would like to migrate my BaNaNaS account to GitHub.

OpenTTD account name: <your account name here>
GitHub profile page: <copy here the URL of the GitHub profile page: https://github.com/*** >

Once we validated you are indeed the author, we will change your content over to your GitHub account. After this, you can edit and upload new content.

Note: please never EVER send us your password. We will never ask for it, but you should also never send it to us.

If you have any questions, feel free to drop by on IRC (irc.oftc.net / #openttd), and we will be happy to answer them.

Sorry for the inconvenience.
